# PR フォーマット

PR のタイトルと本文はこのフォーマットで書く。リポジトリに `.github/PULL_REQUEST_TEMPLATE.md` がある場合はそちらを優先し、このフォーマットは不足セクションの補完に使う。

## タイトル

- 日本語1行・50字以内。変更の要約を「〜を追加」「〜を修正」「〜に統一」の形で書く
- `feat:` 等のプレフィックスは付けない
- タイトルは PR のタイトル欄のみに書き、本文冒頭に H1 として繰り返さない

## 本文

本文は次の5セクションをこの順で並べたものとする:

```markdown
## 概要

何を・なぜを2〜3文で。

## 変更内容

- 変更点の要点を箇条書き（1項目1行目安）
- 実装上の判断があればその理由も

## 背景・経緯

課題や動機。チケット / issue へのリンク。

## 動作確認

- 実行したテスト・確認手順と結果

## レビュー観点

- 特に見てほしい箇所・自信がない箇所
- 議論したい設計判断
```

## ルール

- 変更内容は要点の箇条書きにする。diff の詳細は PR の Files タブで読めるので、本文には判断と構造だけを書く
- 省略できるのは「背景・経緯」（自明な場合）と「レビュー観点」（特にない場合）の2つのみ
- 「動作確認」は必須。何も実行していない場合は「なし（ドキュメントのみの変更のため）」のように理由付きで書く。実行していない確認を書かない
- 本文は日本語で書く
- Draft PR でも本文は最初から書く
