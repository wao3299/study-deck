# 「答えを見る」ボタン 設計

日付: 2026-07-13

## 目的

わからない問題に対して「当てずっぽうで回答する」以外の選択肢がなく、
素直に答えと解説を確認する導線がないギャップを埋める。

## 挙動

- quiz 画面の下部 nav（「回答する」ボタンの並び）に ghost スタイルの
  「答えを見る」ボタンを常設する
- 未回答時に押すと、選択なしのまま採点処理を実行する
  - 全選択肢が正誤色分けされ、選択肢別解説とまとめ解説が表示される
  - **統計には不正解として記録される**（苦手中心モードで優先的に再出題される）
- 押した後は通常の不正解時と同じフロー（「次へ →」で進む）
- 回答済み（採点後）はボタンを隠す
- 本番モードでも表示する（現状の本番モードは即時採点・解説表示ありのため一貫）

## 実装方針

既存の `grade()` は `selected` が空でもそのまま正しく動く
（空選択と正解セットの完全一致比較で必ず不正解になり、全選択肢の
正誤表示・解説表示・統計記録まで一通り行われる）。
「答えを見る」ボタンのハンドラは未回答チェックの上で `grade()` を
呼ぶだけでよく、採点ロジックの変更は不要。

## 変更ファイル

| ファイル | 変更 |
|---|---|
| `src/scripts/quiz.js` | 「答えを見る」ハンドラ、`render()` での表示・採点後の非表示 |
| `src/components/Quiz.astro` | nav に `<button class="btn ghost" id="reveal">答えを見る</button>` を追加 |

## 変更しないもの

- 問題データ形式（`public/questions-*.json`）
- localStorage キー・統計スキーマ（`quizStats_<code>_v1` / `quizHistory_<code>_v1`）
- Firebase 同期（`sync.js` / `merge.js`）— 統計の書き込み経路が変わらないため
- `scripts/validate.js`

## 検証

- `bun run build` が通ること
- `bun run dev` で以下を手動確認する
  - 単一選択・複数選択それぞれで「答えを見る」→ 不正解表示・解説表示・統計反映
  - 「答えを見る」後に苦手中心モードで該当問題の優先度が上がること
  - キーボード操作（A–E / Enter）が従来どおり動くこと
